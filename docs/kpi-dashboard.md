## 📊 KPI 指标体系

> 五大维度，20 项指标。每项有明确阈值、检查方式、扣分规则。满分 100 分，低于 60 分 = 红灯警告。

---

### 一、行为合规 KPI（权重 25%）

| # | 指标 | 目标值 | 检查方式 | 扣分 |
|---|------|--------|----------|------|
| 1.1 | 禁忌违反次数/会话 | **0 次** | 自查回复中是否有 ⚠️标记 | 每违反一条 -5 分 |
| 1.2 | Skill 加载率 | **100%** | 相关 skill 是否在任务前加载 | 每次未加载 -3 分 |
| 1.3 | 工具拦截后重试率 | **≥90%** | 被 BLOCKED 后是否尝试了 ≥3 种替代方案 | 每放弃一次 -5 分 |
| 1.4 | 中文回复率 | **100%** | 检查回复语言 | 每次英文回复 -2 分 |

### 二、架构健康 KPI（权重 25%）

| # | 指标 | 目标值 | 检查方式 | 扣分 |
|---|------|--------|----------|------|
| 2.1 | 子 Agent 使用率 | **≥80%** (长任务中使用 delegate_task) | 统计 delegate_task 调用次数 vs 直接 terminal | 每少用一次 -2 分 |
| 2.2 | 主 Agent 响应速度 | **<5 秒** | 用户发消息到首次回复的间隔 | 每次超时 -2 分 |
| 2.3 | 子 Agent 成功率 | **≥95%** | 子 Agent 完成/失败的比例 | 每失败一次 -3 分 |
| 2.4 | 并行任务利用率 | **≥50%** (可并行任务是否用了 batch) | 检查是否有多子 Agent 并行 | 每次未并行 -2 分 |

### 三、服务可用性 KPI（权重 20%）

| # | 指标 | 目标值 | 检查方式 | 扣分 |
|---|------|--------|----------|------|
| 3.1 | Flask 后端可用率 | **≥99%** | `curl :5000/api/health` | 每次宕机 -10 分 |
| 3.2 | Nginx 可用率 | **≥99%** | `curl -o /dev/null -w '%{http_code}' https://cccarolyn.top/` | 每次非200 -10 分 |
| 3.3 | DeepSeek 代理可用率 | **≥90%** | `curl :8787/v1/models` | 每次宕机 -3 分 |
| 3.4 | ChromaDB 健康 | **向量数 ≥ 知识库条目数** | `/api/vectordb/stats` | 不匹配 -5 分 |

### 四、任务持久化 KPI（权重 15%）

| # | 指标 | 目标值 | 检查方式 | 扣分 |
|---|------|--------|----------|------|
| 4.1 | 跨会话任务续接率 | **100%** | 新会话是否通过 memory + skill 恢复上下文 | 每次丢失 -10 分 |
| 4.2 | Cron Job 存活率 | **100%** | 定时任务是否按计划执行 | 每次漏执行 -5 分 |
| 4.3 | Memory 利用率 | **30-70%** | memory 占用比 (太满需清理，太空没用) | 偏离范围 -2 分 |
| 4.4 | Skill 更新及时性 | **任务完成后 5 分钟内** | 新发现是否及时 patch 到相关 skill | 每次延迟 -2 分 |

### 五、部署质量 KPI（权重 15%）

| # | 指标 | 目标值 | 检查方式 | 扣分 |
|---|------|--------|----------|------|
| 5.1 | Git Push 成功率 | **100%** | commit 后是否能成功 push | 每次失败 -5 分 |
| 5.2 | 部署后验证通过率 | **100%** | 部署后 30 秒内 curl 验证 | 每次未验证 -5 分 |
| 5.3 | 前端 API 对接率 | **100%** | 所有页面功能是否对接真实 API | 每漏一个 -3 分 |
| 5.4 | commit 规范性 | **≥90%** | commit message 是否含 feat/fix/docs 前缀 | 每次不规范 -1 分 |

---

## 📈 评分标准

| 分数 | 等级 | 含义 |
|------|:----:|------|
| 90-100 | 🟢 优秀 | 一切正常，可继续开发 |
| 75-89 | 🟡 良好 | 有小问题，注意改进 |
| 60-74 | 🟠 警告 | 多项不达标，需要纠正 |
| <60 | 🔴 危险 | 严重失职，立即停止并修复 |

---

## 🔍 检查方式

### 手动检查（每次任务开始）

```bash
# 服务健康
curl -s http://127.0.0.1:5000/api/health
curl -s -o /dev/null -w '%{http_code}' https://cccarolyn.top/ai-policy-advisor/
curl -s http://127.0.0.1:5000/api/vectordb/stats

# DeepSeek 代理
curl -s http://127.0.0.1:8787/v1/models

# Git 状态
cd /root/eerobot && git status --short && git log --oneline -3
```

### 自动化检查（Cron 每小时）

通过 cron job 自动运行 KPI 脚本并报告异常。

---

## 🚨 告警规则

| 触发条件 | 动作 |
|----------|------|
| 任何一项 KPI 连续 2 次不达标 | 主动告知用户 + 提出修复方案 |
| Flask 后端宕机 | 立即重启 (systemd 自动) + 通知用户 |
| 总分 < 60 | 暂停新任务，优先修复 KPI |
| 禁忌违反 ≥ 3 次/会话 | 保存为 skill 强化记忆 |
